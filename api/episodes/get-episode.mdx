## Overview

This endpoint returns detailed information for a specific episode identified by the anime's MyAnimeList ID and episode number.

## Authentication

<Note>This endpoint does not require authentication.</Note>

## Rate Limiting

This endpoint is protected by rate limiting middleware:
- **Default limit**: 100 requests per 60 seconds
- Returns `429 Too Many Requests` when limit is exceeded
- Includes `Retry-After` header in rate limit responses

## Query Parameters

<ParamField query="mal_id" type="string" required>
  The MyAnimeList (MAL) ID of the anime. This parameter is required to identify which anime the episode belongs to.
  
  **Example**: `52991`
</ParamField>

<ParamField query="ep" type="string" required>
  The episode number to retrieve. This parameter is required.
  
  **Example**: `1`
</ParamField>

## Response

<ResponseField name="data" type="object">
  Episode object containing detailed information
  
  <ResponseField name="id" type="string">
    Unique identifier for the episode
  </ResponseField>
  
  <ResponseField name="anime_mal_id" type="number">
    MyAnimeList ID of the parent anime
  </ResponseField>
  
  <ResponseField name="episode_id" type="number">
    Episode number within the series
  </ResponseField>
  
  <ResponseField name="video_url" type="string">
    URL to the video stream for this episode. Can be used with video players like Video.js.
  </ResponseField>
  
  <ResponseField name="image_url" type="string">
    Thumbnail or preview image URL for the episode
  </ResponseField>
  
  <ResponseField name="title" type="string" optional>
    Episode title (if available)
  </ResponseField>
  
  <ResponseField name="description" type="string" optional>
    Episode description or synopsis (if available)
  </ResponseField>
  
  <ResponseField name="date" type="string" optional>
    Air date or release date of the episode in ISO format (if available)
  </ResponseField>
</ResponseField>

```typescript TypeScript
interface AnimeEpisode {
  id: string;
  anime_mal_id: number;
  episode_id: number;
  video_url: string;
  image_url: string;
  title?: string;
  description?: string;
  date?: string;
}

const fetchEpisode = async (malId: string, episodeNumber: string) => {
  const response = await fetch(
    `https://anidev.vercel.app/api/episodes/getEpisode?mal_id=${malId}&ep=${episodeNumber}`
  );
  const result: { data: AnimeEpisode } = await response.json();
  return result.data;
};

// Usage
const episode = await fetchEpisode('52991', '1');
console.log(episode.title);
```

```javascript React Example
import { useState, useEffect } from 'react';

function EpisodePlayer({ malId, episodeNumber }) {
  const [episode, setEpisode] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchEpisode = async () => {
      try {
        const response = await fetch(
          `/api/episodes/getEpisode?mal_id=${malId}&ep=${episodeNumber}`
        );
        const result = await response.json();
        
        if (response.ok) {
          setEpisode(result.data);
        } else {
          setError(result.error);
        }
      } catch (err) {
        setError('Failed to fetch episode');
      } finally {
        setLoading(false);
      }
    };

    fetchEpisode();
  }, [malId, episodeNumber]);

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div>
      <h2>{episode.title}</h2>
      <p>{episode.description}</p>
      <video src={episode.video_url} controls />
    </div>
  );
}
```

## Error Responses

### Missing MAL ID Parameter

**Status Code**: `400 Bad Request`

```json
{
  "error": "Missing mal_id parameter",
  "type": "validation"
}
```

### Missing Episode Number

**Status Code**: `400 Bad Request`

```json
{
  "error": "Missing episode number",
  "type": "validation"
}
```

### Episode Not Found

**Status Code**: `404 Not Found`

```json
{
  "error": "Episode not found",
  "type": "notFound"
}
```

### Rate Limit Exceeded

**Status Code**: `429 Too Many Requests`

```json
{
  "error": "Too many requests, please try again later",
  "type": "tooManyRequests"
}
```

**Status Code**: `500 Internal Server Error`

```json
{
  "error": "Internal server error"
}
```

## Implementation Details

<Note>
  This endpoint uses Redis caching to improve performance. Episode data is cached with a configurable TTL to reduce database queries and improve response times.
</Note>

<Warning>
  The `video_url` field may return streaming URLs that require proxy handling. Use the `/api/videoProxy` endpoint if you encounter CORS issues when streaming videos.
</Warning>

## Use Cases

### Video Player Integration

This endpoint is designed to work seamlessly with video players like Video.js:

```javascript
import videojs from 'video.js';

const initPlayer = async (malId, episodeNumber) => {
  const response = await fetch(
    `/api/episodes/getEpisode?mal_id=${malId}&ep=${episodeNumber}`
  );
  const { data } = await response.json();

  const player = videojs('video-player', {
    sources: [{
      src: data.video_url,
      type: 'application/x-mpegURL'
    }],
    poster: data.image_url
  });
};
```

### Progress Tracking

Combine with user preferences to track watching progress:

```javascript
const trackEpisodeProgress = async (malId, episodeNumber) => {
  const episode = await fetchEpisode(malId, episodeNumber);
  
  // Save to user's watch history
  await fetch('/api/user/watchHistory', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      anime_mal_id: episode.anime_mal_id,
      episode_id: episode.episode_id,
      timestamp: Date.now()
    })
  });
};
```
